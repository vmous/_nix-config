# Commit Workflow

When asked to create a commit (or when it's clear a commit is needed):

1. Use `~/.git-commit-template` as the structural template
2. Inspect recent commit messages in the current repo for style/tone
3. For the message content:
   - Describe the change and its effect concisely
   - Avoid duplication across sections
   - Include decisions and tradeoffs from the conversation where applicable
4. Present the proposed commit message for review before committing — do NOT commit until I approve or request modifications
